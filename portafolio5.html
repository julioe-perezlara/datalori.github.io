¿Cuál fue el reto empresarial?

El proceso para visualizar las ordenes de compra (PO) de Walmart que entraron por proceso EDI y que se encontraban en error estaba tardando demasiados días, incluso semanas en Novamex.
Esto estaba sucedíendo por varios motivos:

  • No existía una herramienta o lugar de consulta para las transacciones de Walmart con error.
  • Se requería bastante tiempo para observar las ordenes de compra en error, puesto que el equipo debía primero recordar este proceso, luego ejecutar una serie de queries en SQL y finalmente solucionar o escalar las PO con error

    • No visualizar de inmediato las PO con error estaba impactando negativamente en la relación con Walmart y generando multas y costos extra, puesto que los productos de Jarritos no estaban siendo entregados en la fecha determinada por la transaccion.

    ¿Cómo lo solucionamos?

    Para atacar este reto, SQL Server Management Studio, Excel y Power Query se han utilizado con el fin de desarrollar una aplicación inteligente que permite recuperar las PO con error desde la base de datos, e importarlas hacia una plantilla automatizada Excel.

    Este template produce un reporte actualizado cada que es abierto y muestra todas las ordenes de compra con error existentes en la base de datos.

    ¿Cual fue el resultado?

    Después de 4 sesiones, el profesionista fue capaz de diseñar, automatizar y utilizar las tablas pivote en Excel.

    Esto le ha permitido no solo entender su proceso a través de los datos y detectar problemas, tendencias u oportunidades, sino también presentar a su superior nuevas propuesta de soluciones e implementar algunas mejoras en sus actividades relacionadas al procesamiento de los datos.
