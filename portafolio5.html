¿Cuál fue el reto empresarial?

El proceso para visualizar las ordenes de compra (PO) de Walmart que entraron por proceso EDI y que se encontraban en error estaba tardando demasiados días, incluso semanas en Novamex.
Esto estaba sucedíendo por varios motivos:

  • No existía una herramienta o lugar de consulta para las transacciones de Walmart fallidas.
  • Se requería bastante tiempo para observar las ordenes de compra en error, puesto que el equipo debía primero recordar este proceso, luego ejecutar una serie de queries en SQL y finalmente, solucionar o escalar las PO fallidas

    • No visualizar de inmediato las ordenes de compra con error estaba impactando negativamente en la relación con Walmart y generando quejas y molestias, puesto que los productos de Jarritos no estaban siendo entregados en la fecha determinada por la transaccion.

    ¿Cómo lo solucionamos?

    Para atacar este reto, SQL Server Management Studio, Excel y Power Query se han utilizado con el fin de desarrollar una aplicación inteligente que permite recuperar las PO con error desde la base de datos, e importarlas hacia una plantilla automatizada Excel.

    Este template produce un reporte actualizado cada que es abierto y muestra todas las ordenes de compra fallidas.

    ¿Cual fue el resultado?

    Jarritos Novamex hoy cuenta con un proceso automatizado y una herramienta para visualizar las ordenes de compra que entraron por EDI y que se encuentran en error.
    Esto ha permitido que el equipo de EDI supervise diaramente las PO con error y corregirla, o escalarlas rapidamente a otros departamentos para su pronta resolución.

    Asimismo, utilizar un formato conocido por la mayoría del staff (Excel) y que se actualiza solo, ha permitido:

  • el involucramiento de otros departamentos, incluso llegando al escenario donde ellos también abren el reporte excel y buscaban la solución
  • reducir las quejas de Walmart y mejorar la relación
